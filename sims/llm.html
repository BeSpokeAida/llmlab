Dentro un LLM: dal token alla generazione

Animazione didattica del flusso: tokenizer → embedding → Transformer → logits → prossimo token → ciclo autoregressivo.

Ciclo: 1/3
Fase: 1

Contesto corrente

Il modello vede una sequenza che cresce token dopo token.

Tokenizer

La frase viene spezzata in token appartenenti al vocabolario del modello.

ID → Embedding Matrix

Ogni token diventa un ID. L'ID seleziona una riga della matrice degli embedding.

Posizione

Agli embedding si aggiunge informazione posizionale: il modello deve sapere l'ordine dei token.

Transformer Stack

Più blocchi Transformer ripetono self-attention, residual, normalizzazione e FFN.

Output generato

Ogni nuovo token viene aggiunto al contesto e usato per prevedere il successivo.

Logits e probabilità

Il vettore finale viene proiettato sul vocabolario. I logits diventano probabilità tramite softmax.

Nota didattica

Le dimensioni mostrate, i numeri e le probabilità sono esempi. Nei modelli reali i vettori hanno migliaia di dimensioni, il vocabolario può avere decine/centinaia di migliaia di token e lo stack può includere molti layer.
